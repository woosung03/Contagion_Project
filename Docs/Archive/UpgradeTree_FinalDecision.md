# UpgradeTree 재설계 — 최종 결정안 (구현 확정판)

조사 → MVP 제안 → 밸런스 검증 3단계 과정을 거쳐 종합한 **실제 구현용 최종안**이다(중간 조사/
제안/검증 문서는 결론이 이 문서로 전부 흡수된 뒤 Docs/Archive 정리 과정에서 삭제됨).
이 문서 이후로는 더 조사·검토하지 않는다. 여기 적힌 것만 구현한다.

**핵심 원칙 — "좋은 아이디어" ≠ "지금 넣을 것".**
조사 문서에는 매력적인 제안이 많지만(숨은 조합 보너스·5축 스탯·이벤트 연계 등), 이번 스코프는
**"데이터만 수정, 로직 무변경"인 MVP + 밸런스 리뷰가 '지금 고치라'고 지목한 수치 보정**까지다.
그 이상은 전부 2단계 또는 폐기로 내린다.

**수정 대상 파일 (확정, 2개뿐)**
1. `Assets/Scripts/Data/DefaultUpgradeTreeFactory.cs` — 45개 노드의 `cost`/`effects` 값 교체
2. `Assets/Scripts/UI/UpgradeTreeView.cs` — `NodeDisplayNames`의 `sym_necrosis` 값 1줄 교체

---

## 1. 최종 확정 사항 (반드시 반영 — MVP 제안 원안 그대로)

밸런스 리뷰가 "구조는 맞다"고 승인한 부분. 수치 조정 없이 제안안 그대로 간다.

- **Symptom 3갈래 성격 분리 유지** — 은신형(발진 계열) / 표준형(기침 계열) / 공격형(구토 계열)
- **은신형 유지** — 감염력 위주, 발각도 최소, 치사율 후반까지 억제
- **표준형 유지** — 감염력·치사율·발각도 균형형 (단독 수치는 §2에서 상향)
- **공격형 유지** — 치사율·발각도 위주, 후반 감염력 감소(대가). 밸런스 리뷰 결론: "약한 게 아니라
  비싸고 늦게 강해짐"이 의도에 부합하므로 **수치 변경 없이 그대로**
- **면역교란 유지** — 발각도 억제가 메인, 대가로 감염력 소폭 감소
- **Resistance 재배선 유지** — 약물내성/백신회피/면역교란 3갈래 역할 분리. `drugResistance`
  단일 반복(12/15) 제거, "스탯 상승 → 인류 대응 지연" 카테고리로 전환 (약물내성 수치만 §2 상향)
- **Transmission 예외 구조 유지** — 1~2티어 순수 감염력, 3티어↑(9개, 60%) 발각도 소폭 부착
- **음수 효과(감염력 페널티) 도입 유지** — Symptom 공격형/최종, 면역교란에 처음 도입
- **organfailure 트레이드오프 유지** — 치사율+0.20 / 발각도+0.12 / **감염력-0.08** (사용자 예시
  "장기부전"과 일치). 고정값 감산 방식 자체는 그대로 두되, 근본 재설계(비율 감산)만 2단계로(§3)
- **비용 곡선 비대칭 유지** — 갈래별 ±1~3 차등(공격형·백신회피 프리미엄, 동물매개·은신형·면역교란
  할인). 일부 수치만 §2에서 조정
- **표시명 변경 유지** — `sym_necrosis` "조직 괴사" → **"만성 병변"** (은신형 소속인데 이름이
  공격형처럼 들리는 혼동 제거). 나머지 44개 이름은 그대로

## 2. 수치 수정 후 적용 (밸런스 리뷰 "지금 고칠 것" 반영)

밸런스 리뷰가 **정적 계산으로 지배(dominance) 관계를 확인**한 2건 + 후반 대가 체감 1건.
전부 숫자 리터럴 교체라 구조·로직 변경 없음. 구현 시 MVP 제안 표의 값 대신 아래 값을 쓴다.

### 2-1. 약물내성 상향 — 백신회피의 "거의 지배" 해소 (리뷰 TOP10 #1, 심각도 높음)

문제: 백신회피가 총량 +32%·효율 +5%로 약물내성을 거의 지배 → 약물내성이 함정 선택지.
조치: 약물내성 갈래 총 약물내성을 **0.28 → 0.32**로 끌어올려 효율을 백신회피(0.01088) 위로 역전.

| id | MVP 제안값 | **최종 확정값** |
|---|---|---|
| abl_hardening2 | 약물내성+0.07 | **약물내성+0.09**, 감염력+0.01, 발각도+0.01 |
| abl_resist4 | 약물내성+0.08 | **약물내성+0.10**, 감염력+0.01, 발각도+0.02 |

> 백신회피 비용 추가 상향(34→37)은 이번엔 넣지 않는다. 리뷰 지침이 "둘 중 하나만 해도 완화"이고,
> 두 조치를 동시에 하면 과교정 위험. 약물내성 총량 상향만으로 역전되므로 이쪽만 확정. 백신회피
> 비용 상향은 플레이테스트에서 여전히 밀리면 그때 적용(§3 아님, 플레이테스트 후 미세조정 항목).

### 2-2. 표준형 상향 — 은신형의 지배 완화 (리뷰 TOP10 #2, 심각도 높음)

문제: 은신형이 표준형 대비 감염력/DNA +66%·발각도 부담 절반 → 표준형이 "보람 없는 통행세 브랜치".
조치: 표준형 감염력 소폭 상향 **+** 은신형 저티어 할인 축소로 양쪽에서 격차를 좁힌다.

| id | MVP 제안값 | **최종 확정값** | 방향 |
|---|---|---|---|
| sym_fever | 감염력+0.04, 발각도+0.04, 치사율+0.01 | 감염력**+0.05**, 발각도+0.04, 치사율+0.01 | 표준형 ↑ |
| sym_pneumonia | 감염력+0.05, 치사율+0.05, 발각도+0.06 | 감염력**+0.06**, 치사율+0.05, 발각도+0.06 | 표준형 ↑ |
| sym_lesion | 비용 3 (할인) | 비용 **4** (할인 축소) | 은신형 상대 약화 |
| sym_dermatitis | 비용 5 (할인) | 비용 **6** (할인 축소) | 은신형 상대 약화 |

> 은신형은 **효과는 제안 그대로 두고 비용만 정상화**한다(원래 4→3, 6→5 할인이었던 것을 4/6으로).
> 은신형 갈래 총비용 28 → 30이 되어 감염력/DNA 효율이 내려가고, 표준형 감염력 +0.02 상향과
> 맞물려 두 갈래 격차가 좁혀진다.

### 2-3. 후반 대가 체감 보정 — organfailure 트레이드오프 (리뷰 TOP10 #6, 심각도 중간)

문제: Transmission 풀투자로 감염력이 clamp 1.0 근처면 organfailure의 -0.08 페널티가 체감 무의미.
조치(지금 가능한 최소): 최종 노드 진입 직전 합류 노드의 감소분을 키워 후반 누적 감산이 clamp
안에서도 느껴지게 한다. **근본 해법(고정 감산→비율 감산)은 로직 변경이라 2단계(§3)로 이관.**

| id | MVP 제안값 | **최종 확정값** |
|---|---|---|
| sym_multiorgan2 | 감염력-0.02, 치사율+0.10, 발각도+0.09 | 감염력**-0.03**, 치사율+0.10, 발각도+0.09 |

> `sym_multiorgan1`(감염력+0.03)과 `sym_organfailure`(감염력-0.08)는 제안값 유지.

**§2 요약**: 리터럴 6곳 교체(약물내성 2 · 표준형 2 · 은신형 비용 2 · multiorgan2 1 = 실제 7개 값).
전부 `DefaultUpgradeTreeFactory.cs` 한 파일 안. 구조·비용공식·UI 무영향.

## 3. 2단계로 이관 (지금 넣지 않음 — 로직/신규 필드 필요)

"좋은 아이디어"이지만 데이터 수정 범위를 넘어(코드 로직·신규 스탯 필드) MVP에 못 넣는 것.
1단계 반영·플레이테스트 후 착수 여부를 다시 결정한다.

- **시기 게이팅 (Cure Requirement 대응)** — 최우선 2단계 항목. `UpgradeNode`에 `minCureProgress`/
  `minVisibility` 필드 + `UpgradeManager.CanUnlock()` 조건 1줄. 밸런스 리뷰 결론: **1단계 수치
  조정만으로는 "정답 트리"를 완화만 할 뿐 완전히 없애지 못하며, 이 게이팅이 들어와야 "권장 순서"가
  "강제 순서"가 되어 표준형·약물내성의 존재 의미가 생긴다.**
- **의료 시스템 부담 스탯** — `Pathogen.medicalBurdenModifier` 신규 필드 + `ApplyEffectsToPathogen`
  case + `SimulationManager` 의료부하 계산식에 곱 반영 + `EffectStatLabel()` 라벨 1줄(4개 파일).
  조사 문서 3절의 "감염력/치사율/발각도/백신속도/의료부담" 5축 비전은 이게 들어와야 완성.
- **환경 저항 재활용** — 소비처(`SimulationManager` `climateModifier`)는 이미 있고 노드가 그 값을
  안 건드리는 게 유일한 공백. `ApplyEffectsToPathogen`에 case 1개 추가.
- **국가별 효과 차등** — `SimulationManager` 사망/전파 계산에 `country.developmentLevel` 분기(예:
  약물내성 노드는 선진국에서만 실효). 밸런스 리뷰: 이게 들어와야 "은신형+면역교란"의 무조건 우월이
  상황(국가 상태)에 따라 흔들린다.
- **organfailure 근본 재설계** — 고정 감산(-0.08) → "현재 감염력의 일정 비율 감산". 고정 가산
  구조를 비율 감산으로 바꾸는 로직 변경이라 데이터 범위 밖. §2-3은 임시 완화일 뿐.
- **갈래 시각 태그 UI** — 노드 카드에 "은신형/표준형/공격형" 태그 표시. `UpgradeTreeView`
  레이아웃 변경 필요라 1단계(데이터만) 범위 밖.

## 4. 폐기 (현 로드맵에서 제외 — 구현 대비 가치 낮음)

조사 문서 6절의 3단계(장기) 신규 시스템. 매력적이지만 이번 프로젝트 스코프에서는 **구현 비용 대비
가치가 낮거나 불확실**해 활성 로드맵에서 내린다. 1·2단계 완료 후에도 필요성이 재확인될 때만 재검토.

- **숨은 조합 보너스 (Symptom Combo 대응)** — 조합 감시 시스템 + 신규 조합 정의 데이터가 필요한
  중~고난도 작업인데, **합류 노드(`superbug1` 등)가 이미 "두 갈래 장점을 모으는" 조합 효과를
  대체**하고 있어 별도 시스템의 한계효용이 낮다. (밸런스 리뷰 #9가 오히려 superbug1을 저비용
  고효율로 지목 — 조합 보너스를 더 얹으면 밸런스만 악화)
- **연구 시너지 수치화** — 카테고리 간 상호 견제를 명시적 배율로 공식화. 밸런스 리스크 높고 효과
  불확실. 플레이테스트 선행 없이는 순손해.
- **이벤트 연계** — 노드/조합 해금 시 뉴스 이벤트 트리거. 게임성 핵심이 아닌 연출 항목. 우선순위
  최하위(조사 문서에서도 P3).
- **동적 변이 (원작 Mutation 대응)** — 자동 무작위 특성 부여. 게임 정체성을 바꾸는 리스크(조사
  문서 P4·보류). 현 방향(플레이어 선택 중심)과 상충 → 폐기.

## 5. 최종 작업 순서 — "내가 지금 실제로 구현한다면"

1. **`DefaultUpgradeTreeFactory.cs` — Transmission 9개 노드**: 3티어↑에 발각도 부착 + 갈래 비용
   차등(동물매개 할인 등). MVP 제안 §3 표 그대로.
2. **Symptom 15개 `effects`/`cost` 교체**: MVP 제안 §4 표 기준, 단 **§2-2 상향치 적용**
   (fever 감염력 0.05, pneumonia 감염력 0.06) + §2-3(multiorgan2 감염력 -0.03).
3. **Resistance 15개 재배선**: MVP 제안 §5 표 기준, 단 **§2-1 상향치 적용**
   (hardening2 약물내성 0.09, resist4 약물내성 0.10).
4. **비용 최종 정합**: 은신형 할인 축소(lesion 3→4, dermatitis 5→6, §2-2) 반영, 나머지 프리미엄/
   할인은 제안값 유지. 값 교체 후 갈래별 총비용 재확인(은신형 30 / 표준형 31 / 공격형 35).
5. **`UpgradeTreeView.cs` — `NodeDisplayNames`**: `sym_necrosis` 값 "조직 괴사" → "만성 병변" 1줄.
6. **컴파일 + 에디터 스모크 테스트**: 스탯 clamp(0~1) 정상, 미지원 stat 경고 로그 없음, 음수 효과
   적용 정상 확인. (`Docs/QA_Checklist.md`에 UpgradeTree 항목 추가)
7. **플레이테스트**: ① 은신형+면역교란 조합이 여전히 압도적인지 ② 표준형/약물내성이 이제 경쟁력
   있는지(§2 상향 효과) ③ 후반 organfailure 대가가 체감되는지.
8. **플레이테스트 결과로 미세조정**: 리뷰 TOP10 #3·#4·#7~#9(은신형 시너지 강도, 발각도 후반 집중,
   초반 페널티 체감, 비용 차등 강도, superbug1 효율)를 실측 데이터로 보정. 필요 시 백신회피 비용
   34→37 여기서 적용(§2-1 각주).
9. **2단계 착수 결정**: §3 항목 중 **시기 게이팅 최우선**. 밸런스 리뷰가 "완전한 정답 트리 해소는
   2단계 필요"라 명시했으므로, 1단계가 안정되면 바로 검토.

## 부록 — 결정 근거 한 줄 요약

| 결정 | 근거 |
|---|---|
| 지금 상태로 먼저 구현이 아니라 **수치 보정 반영 후 구현** | 밸런스 리뷰가 약물내성·표준형 지배를 정적 계산으로 확정 — 플레이테스트 없이도 명백한 함정이라 선반영이 안전 |
| 공격형은 수치 손 안 댐 | 리뷰: "약한 게 아니라 비싸고 늦게 강해짐" = 의도 부합 |
| 시기 게이팅을 MVP에 안 넣음 | 로직·신규 필드 필요라 "데이터만 수정" MVP 범위 밖. 단 완전한 해소엔 필수라 2단계 1순위 |
| 숨은 조합 보너스 폐기 | 합류 노드가 이미 조합 효과 대체 + 밸런스 악화 우려 |
