# Research Database 모바일 정보 밀도 재설계 — 후보 5안 비교

Commit 1(Research Database UI Shell)이 실기기/에디터 확인 없이 구현됐는데, 실제로 보니 카테고리당
브랜치 3~4개·항목 11~15개가 한 화면 스크롤에 전부 인라인으로 쌓여 정보 밀도가 높고 스크롤 피로도가
있다는 피드백을 받았다. **Research Database 철학(트리 캔버스 폐기, 리스트/카드 기반 열람)은
유지**하되, 모바일 세로 화면(1440×3120, 회전 잠김)에 맞는 밀도/탐색 구조를 재검토한다.

**순수 비교·조사 문서 — 코드 변경 없음.** 5개 후보 모두 데이터 계층(`UpgradeNode`/`UpgradeManager`/
`DefaultUpgradeTreeFactory`)과 무관하며, `UpgradeTreeView.cs`/`UpgradeTree.uxml`/`UpgradeTree.uss`
UI 레이어만 다시 손댈 대상이다. 실제 착수는 이 문서에서 방향을 정한 다음 별도 커밋.

현재 더미 데이터 기준 규모: 전파 4브랜치/11항목, 증상 4브랜치/14항목, 적응 4브랜치/14항목
(카테고리당 평균 브랜치 4개·항목 3~4개) — 아래 모든 안의 스크롤/밀도 평가는 이 규모를 기준으로 한다.

와이어프레임은 참조 폭 480pt 기준 ASCII 목업. `┃`는 카테고리색 accent-bar, `▔▔▔▔`는 선택 강조.

## A. 인라인 리스트 (Commit 1 현재 구현)

```
┌──────────────────────────────────┐
│ [전파] 증상  적응            DNA:120│
├──────────────────────────────────┤
│ TRANSMISSION LAB                  │

│ TRANS-001 공기전파I    완료   —    │
│ TRANS-002 공기전파II  진행중 18DNA │
│ TRANS-003 비말전파    가능  22DNA │
│ TRANS-004 비말전파강화 잠김  —    │
│┃수인성 계열                       │
│ TRANS-005 수인성전파I  가능 16DNA │
│ TRANS-006 수인성전파II 잠김  —    │
│ TRANS-007 곤충매개전파 잠김  —    │
│┃접촉·동물매개 계열   (스크롤 계속)│
│ ...                                │
├──────────────────────────────────┤
│ 연구 분석 콘솔 / 연구 시작 버튼    │
└──────────────────────────────────┘
```

- **장점**: 항목 도달 2탭(카테고리 탭 + 행 탭). 화면 전환·뒤로가기 없음. 스크롤만 하면 카테고리
  전체 구조(브랜치 4개+선행관계)가 한눈에 보여 "통합 연구"가 어느 브랜치에 의존하는지 위로
  스크롤해서 바로 확인 가능. 구현 완료 상태(변경 비용 0).
- **단점**: 카테고리당 11~15행+헤더 4개가 한 화면에 다 쌓여 스크롤 길다 — 지금 지적된 문제의 원인.
  특정 브랜치를 다시 보려면 스크롤을 되짚어야 함.
- 예상 탭 수(항목 상세까지): **2**
- 예상 스크롤 길이: **김** (11~15행 전부 상시 렌더링)
- 연구소 느낌: 보통 — 리스트형이라 "탐색"보다는 "목록 열람"에 가까움
- Tactical Command Center 적합성: 낮음 — 동시 다중 패널 노출이라는 이 프로젝트 특유의 대시보드
  언어(CountryStatusPanel/CountryPopup의 도넛차트+통계+랭킹 동시 노출)와 결이 다름, 단순 리스트
- 구현 난이도: **완료**

## B. 2단계 드릴다운 (탭 → 브랜치 카드 선택 → 그 브랜치 연구 목록, 별도 화면)

브랜치 선택 화면:
```
┌──────────────────────────────────┐
│ [전파] 증상  적응            DNA:120│
├──────────────────────────────────┤
│ TRANSMISSION LAB                  │
│ ┌──────────────────────────────┐ │
│ │┃공기 계열          2/4 해금 ▶│ │
│ └──────────────────────────────┘ │
│ ┌──────────────────────────────┐ │
│ │┃수인성 계열        0/3 해금 ▶│ │
│ └──────────────────────────────┘ │
│ ┌──────────────────────────────┐ │
│ │┃접촉·동물매개 계열 0/3 해금 ▶│ │
│ └──────────────────────────────┘ │
│ ┌──────────────────────────────┐ │
│ │┃통합 연구          🔒 잠김  ▶│ │
│ └──────────────────────────────┘ │
└──────────────────────────────────┘
```
탭 → 브랜치 연구 목록 화면(별도 스크린, 뒤로가기 있음):
```
┌──────────────────────────────────┐
│ ◀ 공기 계열                DNA:120│
├──────────────────────────────────┤
│ TRANS-001 공기전파I    완료   —    │
│ TRANS-002 공기전파II  진행중 18DNA │
│ TRANS-003 비말전파    가능  22DNA │
│ TRANS-004 비말전파강화 잠김  —    │
├──────────────────────────────────┤
│ 연구 분석 콘솔 / 연구 시작 버튼    │
└──────────────────────────────────┘
```

- **장점**: 두 화면 다 항목이 3~4개뿐이라 스크롤이 거의 없음 — 밀도 문제를 가장 확실하게 해결.
  브랜치 카드에 "2/4 해금" 진행률을 붙이면 지금 인라인 헤더보다 정보가 늘어남. iOS/Android
  설정 앱과 동일한 익숙한 패턴이라 학습 비용 낮음.
- **단점**: 항목 도달 3탭(탭+브랜치+항목)에 화면 전환+뒤로가기까지 필요 — 5안 중 유일하게 실제
  네비게이션 스택이 생긴다. "통합 연구"처럼 여러 브랜치를 선행으로 요구하는 항목의 선행 상태를
  확인하려면 브랜치 화면을 왔다갔다 뒤로가기 해야 해서, A에서는 스크롤 한 번으로 되던 일이 여기선
  더 번거로워짐. 브랜치당 항목 2~4개뿐인 지금 규모에서는 "3개 보자고 화면을 넘어간다"는 탐색
  깊이가 컨텐츠 양 대비 과하다는 인상을 줄 수 있음. 새 화면(뒤로가기 버튼·네비게이션 상태·브랜치
  진행률 계산) 신규 구현 필요 — 5안 중 구현 범위가 가장 큼.
- 예상 탭 수(항목 상세까지): **3** (+ 화면 전환 1회, 뒤로가기 상태 발생)
- 예상 스크롤 길이: **짧음** (브랜치 화면 3~4장, 목록 화면 2~4행 — 둘 다 스크롤 거의 불필요)
- 연구소 느낌: 높음 — 카드형 브랜치 개요가 "연구소 부서를 고른다"는 느낌을 줌
- Tactical Command Center 적합성: **낮음~보통** — 모바일 범용 드릴다운(설정 앱류) 패턴이라
  오히려 "콘솔"보다 "메뉴 앱" 느낌에 가까움. 이 프로젝트가 지금까지 피해온 방향(뒤로가기 스택,
  화면 전환)에 정면으로 부딪힘
- 구현 난이도: **높음** — 신규 화면, 네비게이션 상태, 뒤로가기, 브랜치 진행률 집계 전부 신규

## C. 아코디언 (같은 화면, 브랜치 헤더 탭 시 그 브랜치만 펼침)

```
┌──────────────────────────────────┐
│ [전파] 증상  적응            DNA:120│
├──────────────────────────────────┤
│ TRANSMISSION LAB                  │
│▶┃공기 계열          2/4 해금       │
│▼┃수인성 계열        0/3 해금       │
│   TRANS-005 수인성전파I 가능 16DNA│
│   TRANS-006 수인성전파II 잠김 —   │
│   TRANS-007 곤충매개전파 잠김 —   │
│▶┃접촉·동물매개 계열 0/3 해금       │
│▶┃통합 연구          🔒 잠김        │
├──────────────────────────────────┤
│ 연구 분석 콘솔 / 연구 시작 버튼    │
└──────────────────────────────────┘
```

- **장점**: 탐색 단계를 늘리지 않고(같은 화면, 뒤로가기 없음) 밀도 문제만 정확히 겨냥. 기본
  전부 접힌 상태로 진입하면 브랜치 헤더 4개만 보여 첫인상 밀도가 낮음. "통합 연구" 확인 시에도
  화면 전환 없이 헤더만 펼치면 되므로 A의 "스크롤로 전체 구조 파악" 장점을 거의 그대로 유지.
- **단점**: 접힌 브랜치를 볼 땐 헤더 탭이 한 번 더 필요(기본 펼침 브랜치가 아니면 3탭). 여러
  브랜치를 동시에 펼쳐두면 결국 A와 밀도가 같아짐 — "한 번에 하나만 펼침" 같은 정책을 정해야
  효과가 유지됨(아코디언 정책 미결정 리스크).
- 예상 탭 수(항목 상세까지): 기본 펼침 브랜치는 **2**, 그 외 브랜치는 **3**(헤더 탭 1회 추가)
- 예상 스크롤 길이: **짧음** (접힌 상태 기준 헤더 4개, 펼친 브랜치만 추가로 2~4행)
- 연구소 느낌: 보통~높음 — 헤더에 진행률 배지를 붙이면 나쁘지 않음
- Tactical Command Center 적합성: 보통 — 아코디언 자체는 일반적인 모바일 UI 패턴이라 톤은
  중립적, 진행률 배지·accent-bar로 기존 톤을 덧입힐 수 있음
- 구현 난이도: **낮음~중간** — 펼침/접힘 상태(카테고리당 활성 브랜치 1개 정도)와 브랜치 요약
  배지 계산만 추가, 신규 화면 없음

## D. 연구 콘솔 (상단 브랜치 세그먼트 선택 + 본문에 활성 브랜치만, 단일 화면)

```
┌──────────────────────────────────┐
│ [전파] 증상  적응            DNA:120│
├──────────────────────────────────┤
│ TRANSMISSION LAB                  │
│ [공기] [수인성] [접촉] [통합]      │  ← 세그먼트 컨트롤(가로 스크롤 가능)
│  ▔▔▔▔                             │
├──────────────────────────────────┤
│ TRANS-001 공기전파I    완료   —    │
│ TRANS-002 공기전파II  진행중 18DNA │
│ TRANS-003 비말전파    가능  22DNA │
│ TRANS-004 비말전파강화 잠김  —    │
├──────────────────────────────────┤
│ 연구 분석 콘솔 / 연구 시작 버튼    │
└──────────────────────────────────┘
```

- **장점**: 화면 전환 없이 활성 브랜치 항목(2~4개)만 렌더링돼 스크롤이 거의 사라짐. 세그먼트
  컨트롤은 "채널 선택 다이얼" 같은 조작감이라 HUD/레이더 화면 문법과 잘 맞음 — 5안 중 "연구
  콘솔을 조작한다"는 느낌이 가장 강함.
- **단점**: 세그먼트 라벨이 브랜치명 축약판("공기"/"수인성"/"접촉"/"통합")이라 브랜치 풀네임
  정보가 줄어듦. "통합 연구"의 교차 브랜치 의존성을 확인하려면 세그먼트를 다시 눌러 브랜치를
  옮겨야 함(화면 전환은 없지만 왕복 탭 필요).
- 예상 탭 수(항목 상세까지): **3** (카테고리 탭 + 세그먼트 탭 + 항목 탭), 화면 전환 없음
- 예상 스크롤 길이: **매우 짧음** (활성 브랜치 항목만 렌더링)
- 연구소 느낌: **매우 높음** — 채널/다이얼 조작 은유
- Tactical Command Center 적합성: **높음** — 세그먼트 컨트롤은 기존 HUD 탭 언어의 연장선
- 구현 난이도: **중간** — 세그먼트 컨트롤 신규 + "활성 브랜치" 상태 추가, 신규 화면·뒤로가기 없음

## E. 브랜치+연구 콘솔 (마스터-디테일 동시 노출 — 브랜치 현황판 + 리스트, 상시 표시)

```
┌──────────────────────────────────┐
│ [전파] 증상  적응            DNA:120│
├──────────────────────────────────┤
│ TRANSMISSION LAB                  │
│┏━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━┓│
│┃▶공기 계열      2/4 ██░░       ┃│  ← 선택됨(강조), 진행률 바
│┃ 수인성 계열    0/3 ░░░        ┃│
│┃ 접촉·동물매개  0/3 ░░░        ┃│
│┃ 통합 연구      🔒 ░░░         ┃│
│┗━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━┛│
├──────────────────────────────────┤
│ TRANS-001 공기전파I    완료   —    │
│ TRANS-002 공기전파II  진행중 18DNA │
│ TRANS-003 비말전파    가능  22DNA │
│ TRANS-004 비말전파강화 잠김  —    │
├──────────────────────────────────┤
│ 연구 분석 콘솔 / 연구 시작 버튼    │
└──────────────────────────────────┘
```

- **장점**: D와 동일하게 화면 전환 없이 활성 브랜치만 리스트에 렌더링되지만, 브랜치 선택자가
  세그먼트 칩이 아니라 **진행률 바 포함 브랜치 현황판**(세로 4행, 상시 표시) — 카테고리 전체
  브랜치의 해금 현황을 항상 한눈에 보여준다. `CountryStatusPanel`의 `distribution-row`/
  진행률 바 언어를 그대로 재사용 가능해 신규 색상 축·컴포넌트 없이 톤이 가장 잘 맞음. "연구소
  현황판"이라는 Research Database 철학을 가장 강하게 구현.
- **단점**: 브랜치 현황판(4행 고정)이 화면 상단 공간을 항상 차지해, D보다 화면 구성 요소가
  많다(세그먼트 한 줄 vs 진행률 패널 4행) — 화면이 더 "꽉 차 보일" 수 있음(다만 각 요소가
  짧아 스크롤 유발 정도는 아님). 브랜치별 해금 개수 집계 로직이 D의 라벨보다 더 필요.
- 예상 탭 수(항목 상세까지): **3** (카테고리 탭 + 브랜치 행 탭 + 항목 탭), 화면 전환 없음
- 예상 스크롤 길이: **짧음** (브랜치 패널 4행 고정 + 활성 브랜치 항목 2~4행, 둘 다 스크롤 불필요)
- 연구소 느낌: **매우 높음** — 진행률 바가 붙은 "연구소 현황판" 느낌

## 권장안

**E(브랜치+연구 콘솔)를 최종 목표로, D(연구 콘솔/세그먼트)를 더 가벼운 대안으로 권장한다.** 근거:

1. 이 프로젝트는 이미 `CountryStatusPanel`/`CountryPopup`에서 "여러 판독 패널을 화면 전환 없이
   동시에 보여주는" Tactical Command Center 문법을 확립했다 — E가 이 문법을 Research Database에
   그대로 계승한다. B(드릴다운)는 정반대로 화면 전환·뒤로가기를 도입해 지금까지 이 프로젝트가
   피해온 패턴에 부딪힌다.
2. 지금 지적된 문제(밀도·스크롤 피로)는 B처럼 화면을 늘리지 않아도 C/D/E 셋 다 해결한다 —
   화면 전환은 문제 해결의 필요조건이 아니다.
3. 브랜치당 항목이 2~4개뿐인 지금 데이터 규모에서, B의 "브랜치 전용 화면"은 콘텐츠 양 대비
   탐색 단계가 과하다. 반대로 E/D는 같은 화면 안에서 즉시 전환되므로 콘텐츠가 적어도 어색하지
   않다.
4. C(아코디언)는 가장 구현 리스크가 낮은 안전한 선택지 — E/D로 가기 전에 우선 C로 밀도 문제만
   빠르게 해소하고, Commit 2(실제 데이터 연결) 이후 브랜치 진행률 집계가 자연스러워지는 시점에
   D/E로 확장하는 단계적 접근도 가능하다.
