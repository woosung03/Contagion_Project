# UpgradeTree 1단계(MVP) 변경 제안 — 승인 대기, 코드 미수정

`Docs/UpgradeTree_Redesign_Investigation.md` 6절의 1단계(MVP) 4개 항목(노드 효과 다중화 /
Resistance 재배선 / 비용 곡선 비대칭화 / 표시명 정리)에 대한 구체적 변경안. **이 문서는 제안이며
`DefaultUpgradeTreeFactory.cs`/`UpgradeTreeView.cs`는 아직 수정하지 않았다.** 승인 후 다음
세션(또는 이어서)에서 실제 코드에 반영한다.

**이번 범위에 포함하지 않는 것** (다음 단계로 보류, 이번 제안에 없음): 시기 게이팅(Cure
Requirement), 의료 시스템 부담 스탯, 환경 저항 재활용, 숨은 조합 보너스.

## 1. 현재 구조 요약 (재확인)

세 카테고리 모두 동일한 위상: **3갈래 × 4티어 → 2개 합류 노드(5티어) → 1개 최종 노드(6티어) = 15개**.

```
갈래A(T1→T2→T3→T4) ─┐
                     ├→ 합류1(T5a) ─┐
갈래B(T1→T2→T3→T4) ─┘              ├→ 최종(T6)
갈래C(T1→T2→T3→T4) ──→ 합류2(T5b) ─┘
```

- **Transmission**: 공기(air→droplet) / 수인성·동물매개(water→animal) / 접촉(contact→insect→blood)
- **Symptom**: 기침 계열(cough→fever→pneumonia→respfailure) / 발진 계열(rash→lesion→dermatitis→
  necrosis) / 구토 계열(nausea→vomit→hemorrhage→sepsis)
- **Ability**: 변이 계열(mutation→resist1,3) / 은신 계열(stealth→camouflage) / 구조강화 계열
  (hardening→resist2,4)

기본 비용은 지금 **세 갈래 모두 티어당 동일**(2/4/6/8, 합류 12, 최종 20) — 이번 3번 항목에서
갈래별로 차등을 준다.

## 5. Ability(Resistance) — 3갈래 재배선 (drugResistance 단순 반복 제거)

기존 "변이/은신/구조강화" 3갈래는 이름만 다르고 실질적으로 12/15개가 `drugResistance` 단일
반복이었다. 아래처럼 역할을 명확히 나눈다.

- **약물내성** (구조강화 계열 재활용: hardening→resist2,4) — 방어 위주, 부수적으로 감염력 소폭 보너스

  비용도 가장 비싸고 발각도 대가가 붙음
- **면역교란** (은신 계열 재활용: stealth→camouflage) — 발각도를 억제하는 게 메인 효과, 대가로
  감염력을 소폭 깎음(원작에 없는 신규 트레이드오프 — "숨으려면 활동성을 낮춰야 한다")

| sym_necrosis | 조직 괴사 | 만성 병변 | "은신형(발진 계열)" 소속인데 "괴사"라는 이름이 공격형처럼 들려 혼동 — 갈래 성격(잠행)에 맞게 완화 |

나머지 44개는 이름 자체가 이미 계열 성격과 크게 안 어긋나 유지한다. 다만 **UI 카테고리 캡션**에
`UpgradeTreeView.CategoryLabel()`/`CategoryCaption()` 근처에 "은신형/표준형/공격형",
"약물내성/백신회피/면역교란" 갈래 구분을 시각적으로 드러내고 싶다면(예: 노드 카드에 갈래 태그
표시) 이는 UI 레이아웃 변경이 필요해 **1단계 범위를 벗어난다** — 이번엔 효과/비용/이름 데이터만
바꾸고, 갈래를 시각적으로 태그하는 것은 별도 UI 작업으로 다음에 논의할 항목으로 남겨둔다.

## 7. 영향도 설명

- **다중효과 비율**: 45개 중 기존 13개(29%)만 다중효과 → 변경 후 Symptom 15/15(100%) + Ability
  15/15(100%) + Transmission 9/15(60%) = **총 39/45(87%)**. Transmission 저티어 6개만 예외로
  남긴다(1절/3절 근거).
- **카테고리 간 상호 의존**: Symptom의 발각도/치사율 상승 대가를 Resistance의 면역교란/백신회피가
  각각 다른 방식으로 상쇄하도록 배선했다 — 한 카테고리만 몰아 찍으면 반드시 손해(발각도 누적 또는
  치료제 경쟁 열세)를 본다.
- **비용 곡선**: 갈래별 최종/후반 노드 비용을 ±1~3 정도로만 차등했다(공격형·백신회피는 프리미엄,
  은신형·면역교란은 할인). 급격한 변화가 아니라 "이 갈래를 끝까지 몰아 찍는 게 항상 최선"이라는
  느낌을 완화하는 수준 — 밸런스 붕괴 리스크를 낮게 유지했다.
- **총 스탯 캡(Clamp01) 영향**: 각 스탯은 여전히 0~1로 클램프되므로(`UpgradeManager.
  ApplyEffectsToPathogen`), 신규 감염력 소폭 보너스/페널티가 누적돼도 극단으로 튀지 않는다.
  다만 Symptom 최종 노드의 감염력 -0.08, Ability 최종 노드의 발각도 -0.03 등 **음수 효과가
  처음 도입**되므로, 순서(어떤 노드를 먼저 사는지)에 따라 최종 스탯이 갈릴 수 있다 — 플레이테스트로
  감염력/치사율 총합이 극단적으로 낮아지는 조합이 없는지 확인이 필요하다(예: 은신형+공격형을
  동시에 깊게 파는 조합에서 감염력이 과도하게 깎이는지).
- **리스크**: 코드 변경량 자체는 작다(모두 `DefaultUpgradeTreeFactory.cs`의 숫자 리터럴 + `Ability`
  effect entry 추가, `UpgradeTreeView.cs` 표시명 딕셔너리 1줄). UI 로직/레이아웃은 무변경이라
  회귀 위험은 낮다. 밸런스 수치(구체적 %)는 초안이며 플레이테스트 후 조정이 필요할 수 있다.
